```text
Build a web app called AnimalView: a Bear 71-inspired animal movement POV player.

Goal:
```

## Backend endpoints

```text
GET /api/species
Returns a curated list of demo species with Movebank study IDs.

GET /api/studies?species=...
Returns Movebank studies matching the species.

GET /api/individuals?study_id=...
Returns individuals from a Movebank study.

GET /api/track?study_id=...&individual_id=...&from=...&to=...
Downloads movement points from Movebank and returns GeoJSON.

POST /api/match-imagery
Input:
{
  "points": [
    {
      "lat": 43.1,
      "lon": 5.9,
      "timestamp": "2024-01-01T12:00:00Z"
    }
  ],
  "radius": 50,
  "providers": ["google", "mapillary"]
}

Output:
{

      "confidence": "medium",

}
```

```sql

  id INTEGER PRIMARY KEY AUTOINCREMENT,
  study_id TEXT,
  individual_id TEXT,

  lon REAL

CREATE TABLE imagery_matches (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  study_id TEXT,
  individual_id TEXT,
  track_point_index INTEGER,
  provider TEXT,
  pano_id TEXT,
  image_id TEXT,

```

## Matching logic

```text
1. Load track points.
2. Downsample points to avoid API abuse:
   - keep one point every 50–200 meters, or one point every N minutes.
3. For each sampled point:
   - Query Google Street View metadata:
     location=lat,lon
     radius=selected radius
     key=GOOGLE_MAPS_API_KEY
   - If status is OK, store panorama metadata.
4. Query Mapillary images near the coordinate using bounding box or proximity search.
5. Compute distance between animal point and image point using Turf.js.
6. Estimate animal heading:
   - bearing from previous track point to next track point.
7. Compute confidence:
   - High: image within 25m
   - Medium: image within 100m
   - Low: image within 250m
8. Sort matches by:
   - nearest distance
   - closest image date to animal timestamp, if available
   - provider availability
```
